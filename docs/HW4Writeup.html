Overview

In this homework, we develop our own cloth simulation that models realistic cloth behavior using point masses and springs. We implement three different physical constraints, structural, shearing, and bending springs, to preserve the shape and flexibility of the cloth. We also handle collisions with objects like spheres and planes to simulate interactions with other objects. We explore how parameters such as the spring constant (ks), density, and damping affect the cloth’s movement and final resting shape. Additionally, we incorporate efficient self-collision detection using a spatial map and enhance the visual effects with shading techniques like Blinn-Phong, bump mapping, displacement mapping, and custom shaders. By experimenting with different physical and visual parameters, we gain a deeper understanding of both the dynamics of cloth simulation and applying rendering in simulation.

Part 3

Implementation

Implementing collisions with spheres and planes involves writing each primitive’s collide functions and handling them in the simulate function. For a sphere, the collide function checks if a point mass is inside the sphere by comparing the distance to the sphere's origin with its radius. If a collision is detected, it projects the point to the sphere’s surface and applies a correction based on the last position and friction. For a plane, the collide function detects if a point mass has crossed the plane between its last and current positions w.r.t the plane  by checking if a sign change happens. If a point mass has crossed the plane, it projects the point onto the side where its last position is on with a small offset and applies a friction  correction. Finally, in simulate function, we loop through all point masses and all objects  to handle collisions.

Implementation of Self-Collision

To implement self-collision, we first build a spatial map that groups nearby point masses using a hash of their positions to speed up. For each simulation step, we build a new spatial map accordingly. During the simulation, for each point mass, we use this map to find nearby points within twice the cloth thickness. If any are too close, we compute and accumulate offset vectors to separate them. The average correction is then scaled by the number of collisions and simulation steps, and applied to the point’s position to prevent overlapping.

Part 5

What is a Shader Program

A shader program runs on the GPU to control how 3D objects are rendered, primarily through vertex shaders and fragment shaders. The vertex shader processes each vertex, transforming positions and preparing data like normals or texture coordinates. This data is then interpolated and passed to the fragment shader, which computes the final color of each pixel by applying lighting and material effects (like shading, textures, or reflections).

Blinn-Phong Shading Model

The Blinn-Phong shading model combines three shading components: ambient (constant background light), diffuse (light scattered in many directions based on surface angle), and specular (sharp reflections that depend on the viewer’s position). Instead of using the exact reflection vector like Phong, it uses a halfway vector between the light and view directions to compute specular highlights more efficiently. This makes it faster and visually smoother, especially for shiny surfaces.

Bump vs. Displacement Mapping

Comparison: Bump mapping fakes surface texture by altering surface normals in the fragment shader, giving the illusion of depth without changing the actual geometry. Displacement mapping actually modifies the geometry by pushing vertices based on a height map, creating real depth.

When the grid is coarse, using bump mapping technique results in a still smooth surface, whereas displacement mapping would turn the sphere into a polygon, worsening the rendering quality. When the grid is fine, the bump map still gives a smooth overall shape with only an illusion of roughness produced by the fragment shader, whereas displacement mapping can actually produce the surface unevenness in geometric details.
